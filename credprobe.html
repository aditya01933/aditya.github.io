Why Credprobe?

When credentials leak—from git repos, compromised databases, or phishing attacks—you need immediate answers: Is this credential still active? What access does it have? Which service is it for?

Credprobe validates credentials directly against provider APIs, giving you definitive answers in seconds.

Features

Multi-Provider

130+ services including GitHub, AWS, Stripe, Slack, and every major API platform

Rate Limited

Built-in per-provider rate limiting prevents API abuse and bans

Extensible

Add new providers via YAML configs or Ruby validators in minutes

Rich Metadata

Returns user info, scopes, permissions, and access levels

Thread-Safe

Concurrent validation for high-throughput security operations

Production Ready

Zero dependencies beyond Faraday and Nokogiri

Quick Start

gem install credprobe
require 'credprobe'

result = Credprobe.validate(
  secret: 'ghp_1234567890abcdef',
  provider: :github
)

if result[:active]
  puts "⚠️  ACTIVE TOKEN"
  puts "User: #{result[:metadata][:user]}"
  puts "Scopes: #{result[:metadata][:scopes]}"
else
  puts "✅ Inactive"
end

YAML DSL

Define new validators without writing Ruby code. Drop a YAML file in config/providers/ and it auto-loads.

View YAML DSL documentation

Basic Structure

url: https://api.example.com/validate
method: POST
timeout: 10

headers:
  Authorization: "Bearer {{secret}}"

response_mapping:
  - status: 200
    body_type: json
    body_contains:
      - json_path: "authenticated"
        equals: true
    result: active
  
  - status: 401
    result: inactive

rate_limit:
  requests: 100
  window: 3600

Template Variables

Use {{secret}} and {{metadata.key}} anywhere in URLs, headers, or request bodies.

Body Conditions

body_contains:
  - contains: "success"              # String search
  - json_path: "data.user.active"    # JSON path
    equals: true
  - json_path: "errors[0].code"      # Array access
  - regex: "error.*rate.?limit"      # Regex match
  - xpath: "//Error/Code"            # XML XPath
    equals: "AccessDenied"

Status Matching

response_mapping:
  - status: 200              # Exact match
  - status: "400-499"        # Range match
  - status: [401, 403]       # Multiple values
⚠️ Security Notice

Credprobe makes real API calls to validate credentials. Use in secure environments only. Some providers may log validation attempts. Logs are automatically sanitized but consider using dedicated validator accounts where possible.
