📊 The Research That Started It All

In June 2024, researchers from University of Texas at San Antonio, Virginia Tech, and University of Oklahoma published the first comprehensive analysis. They tested 16 different LLMs across two massive prompt datasets, generating 576,000 code samples.

🛠️ Build Your Own Detection in 15 Minutes

Since most security tools don't cover this yet, here's a lightweight scanner you can deploy today:

# hallucination_detector.py
import requests
import sys

def check_package_exists(package_name, ecosystem='pypi'):
"""Query registry API to verify package exists"""
if ecosystem == 'pypi':
url = f"https://pypi.org/pypi/{package_name}/json"
else: # npm
url = f"https://registry.npmjs.org/{package_name}"

try:
response = requests.get(url, timeout=10)
return response.status_code == 200
except:
return False

def scan_requirements(file_path):
"""Scan requirements.txt for hallucinated packages"""
with open(file_path) as f:
packages = [line.split('==')[0].strip()
for line in f if line.strip() and not line.startswith('#')]

hallucinated = []
for pkg in packages:
if not check_package_exists(pkg):
hallucinated.append(pkg)

return hallucinated

# Usage in GitLab CI:
# python hallucination_detector.py requirements.txt

📖 The Deeper Technical Details

Want the full research breakdown? Key findings from the validated studies:

Temperature Settings Matter

Lowering LLM temperature from 1.0 to 0 (deterministic output) reduced hallucination rates by 30-40%. But this also reduced code creativity and diversity. The tradeoff: safer but more repetitive suggestions.

Three Categories of Hallucination

38% - Inspired by real packages (requests-async looks like aiohttp)
13% - Typos/transpositions (character-level mistakes)
51% - Pure fabrication (completely invented based on learned patterns)

Better Coders = Fewer Hallucinations

January 2025 research found a strong inverse correlation (r = -0.80) between HumanEval coding scores and hallucination rates. Models that write better code also hallucinate fewer phantom packages.
